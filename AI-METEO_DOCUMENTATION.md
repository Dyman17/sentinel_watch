## Часть 2: Аннотация (краткое описание)

### Суть проекта

Мы создаём сеть умных метеостанций на базе **ESP32 с датчиками BME280/DHT22**, которые не просто измеряют погоду, а **предсказывают локальные осадки с точностью 85% на 3 часа вперёд** с помощью нейросетей (LSTM/Transformer).

Пока Гидрометцентр говорит о погоде в масштабе города, наш **AI-Meteo предупреждает, что именно на вашей улице через час начнётся дождь**. Все данные и прогнозы доступны на интерактивной карте в реальном времени.

Система комбинирует:
- 🌡️ **Наземные IoT-датчики** (температура, влажность, давление)
- 🛰️ **Спутниковые данные** (Sentinel-2, Sentinel-3 для облачности)
- 🧠 **Нейросети** (LSTM для временных рядов)
- 🗺️ **Веб-визуализацию** (интерактивная карта Leaflet)
- 🤖 **Telegram-бот** (быстрый доступ к прогнозам)

### Ключевая решаемая проблема

**Отсутствие точных микро-локальных метеопрогнозов:**
- Существующие прогнозы (Гидрометцентр, MeteoInfo) имеют разрешение 10-100 км
- Они не учитывают локальные особенности микрорайонов, парков, водоёмов
- Люди промокают под дождём, потому что не знают, что именно на их улице через час начнётся ливень
- Строительные компании, event-агентства и коммунальные службы теряют деньги из-за невозможности планировать работы с учётом реальной микро-погоды

### Роль ИИ в решении

AI-Meteo использует глубокое обучение для анализа:
1. **Исторических временных рядов** с датчиков (LSTM анализирует паттерны за последние 24-72 часа)
2. **Спутниковых снимков облачности** (Sentinel для визуального анализа облачных фронтов)
3. **Локальных географических особенностей** (рельеф, растительность, водоёмы)
4. **Взаимодействия между станциями** (Transformer-архитектура для синтеза данных соседних станций)

**Результат:** прогноз вероятности осадков (0-100%) и интенсивности дождя с точностью 85% на горизонт 3 часа.

## Часть 3: Введение

### Актуальность проекта для космической отрасли

AI-Meteo демонстрирует **синергию наземных IoT-датчиков и спутниковых данных** — это ключевая задача современной "последней мили" космических технологий:

- **Калибровка спутниковых данных** через наземные датчики повышает точность анализа облачности
- **Спутниковые снимки** (Sentinel, Meteosat) дают контекст о крупномасштабных облачных фронтах
- **Локальные датчики** заполняют пробел в пространственном разрешении спутниковых данных
- Проект показывает, как космические данные могут быть усилены наземной сетью для решения **прикладных задач "последней мили"** (микрорайоны, конкретные координаты)

Это важно для конкурса AEROO SPACE, так как демонстрирует **практическое применение космических технологий** в решении локальных проблем.

### Целевая аудитория

1. **Жители микрорайонов** — знать, пойдёт ли дождь именно на их улице
2. **Малый бизнес** — кафе с летними верандами, парикмахерские, магазины
3. **Event-агентства** — планировать открытые мероприятия с уверенностью
4. **Строительные бригады** — оптимизировать работы по погодным условиям
5. **Коммунальные службы** — предупредить жителей о потопах, подготовить дренаж
6. **Сельское хозяйство** — прогноз осадков для полива и защиты урожая

### Проблематика (подробно)

#### Существующие решения неэффективны:
- 📊 **Разрешение 10-100 км** — один прогноз на весь город, игнорируя локальные особенности
- ❌ **Не учитывают микроклимат** — парки, водоёмы, плотность застройки влияют на осадки
- 🚫 **Задержка в обновлении** — данные обновляются раз в 3-6 часов
- 💸 **B2B недоступны** — нет API для малого бизнеса (дорого или невозможно)

#### Следствия:
- 🌧️ Люди промокают, теряя продуктивность и здоровье
- 🎪 Event-агентства отменяют мероприятия "на всякий случай"
- 🏗️ Строители останавливают работы без точной информации
- 📱 Полагаются на "народные приметы" или случайные прогнозы

#### AI-Meteo решает:
- ✅ **Разрешение ~ 500 м** (размер микрорайона)
- ✅ **Обновление каждые 15 минут** в реальном времени
- ✅ **Доступно для всех** (веб-сайт + Telegram-бот)
- ✅ **Точность 85%** на горизонт 3 часа

   ↓

## Часть 6: Модель ИИ и алгоритмы

### Используемая архитектура

#### Основная модель: LSTM (Long Short-Term Memory)

```
Архитектура:

Input Layer (shape: [batch_size, timesteps=96, features=7])
  ↓ (96 = последние 24 часа с шагом 15 минут)
  ↓ (features: temp, humidity, pressure, cloud_cover, hour, day_of_week, seasonal_index)

Embedding Layer (опционально, если добавляем категориальные признаки)
  ↓

LSTM Layer 1 (units=128, return_sequences=True, dropout=0.2)
  ↓

LSTM Layer 2 (units=64, return_sequences=False, dropout=0.2)
  ↓

Dense Layer (units=64, activation='relu')
  ↓

Dropout (rate=0.3)
  ↓

Dense Layer (units=32, activation='relu')
  ↓

Output Layer (units=3, activation='sigmoid')
  ↓ (3 выхода: вероятность дождя в t+60мин, t+120мин, t+180мин)

Output shape: [batch_size, 3]
Диапазон: [0, 1] — вероятность осадков в процентах
```

**Почему LSTM?**
- ✅ Работает с временными рядами (последовательности)
- ✅ "Память" о прошлых состояниях (LSTM gate механизм)
- ✅ Предсказывает на несколько шагов вперёд (3 часа)
- ✅ Быстрый инференс (~50мс на CPU)

#### Дополнительная модель: Transformer-based Informer (опционально)

```
Архитектура Informer для временных рядов:

Input: [station_1, station_2, station_3] временные ряды одновременно
  ↓
Embedding Layer (преобразование в 256-мерные векторы)
  ↓
Positional Encoding (информация о временных позициях)
  ↓
Encoder Stack (6 слоёв):
  - Self-Attention (ProbSparse, экономит память)
  - Feed-Forward Network
  - Layer Normalization
  - Dropout
  ↓
Decoder Stack (6 слоёв):
  - Cross-Attention (между кодером и декодером)
  - Self-Attention
  - Feed-Forward
  - Layer Normalization
  ↓
Output Projection (преобразование в прогнозы)
  ↓
Output: [prob_rain_t+60, prob_rain_t+120, prob_rain_t+180]
```

**Преимущества Transformer:**
- ✅ Использует спутниковые данные и данные соседних станций одновременно
- ✅ Параллельная обработка (быстрее LSTM)
- ✅ Лучше захватывает дальние зависимости
- ⚠️ Требует больше памяти и данных для обучения

### Принцип работы модели

```
ФАЗА 1: ПОДГОТОВКА ВХОДНЫХ ДАННЫХ (Feature Engineering)

Сырые данные от ESP32:
  timestamp | station_id | temp | humidity | pressure
  2026-02-26 14:00 | station_1 | 18.5 | 65 | 1013.2
  2026-02-26 14:05 | station_1 | 18.6 | 64 | 1013.1
  ...

↓ Обработка:

1. Извлечение из БД последние 24 часа (96 точек по 15 минут):
   X = [[18.5, 65, 1013.2, ...], [18.6, 64, 1013.1, ...], ...]  # shape: (96, 7)

2. Нормализация (StandardScaler):
   X_scaled = (X - mean) / std  # каждый признак имеет mean=0, std=1

3. Добавление спутниковых данных (облачность из Sentinel):
   cloud_cover = [0.3, 0.35, 0.4, ...]  # 0-1 (0 = ясно, 1 = облачно)

4. Добавление временных признаков:
   hour_of_day = [14, 14, 14, ...]  # 0-23
   day_of_week = [3, 3, 3, ...]  # 0-6 (Wednesday)
   season = [0.7, 0.7, 0.7, ...]  # 0-1 (зима=0, весна=0.33, лето=0.66, осень=1)

5. Стекинг:
   X_final = [temp, humidity, pressure, cloud_cover, hour, day_of_week, season]
   X_final.shape = (96, 7) ← готово для модели

---

ФАЗА 2: ИНФЕРЕНС (Prediction)

X_final (shape: 1, 96, 7) → LSTM Model → [0.15, 0.45, 0.25]

Интерпретация:
  - Вероятность дождя в t+60мин: 15% (малые осадки или нет)
  - Вероятность дождя в t+120мин: 45% (осадки вероятны)
  - Вероятность дождя в t+180мин: 25% (после прохождения фронта)

---

ФАЗА 3: ПОСТОБРАБОТКА (Post-processing)

1. Сглаживание (Savitzky-Golay filter для удаления шума):
   [0.15, 0.45, 0.25] → [0.18, 0.43, 0.27]

2. Калибровка по исторической точности:
   Если модель систематически переоценивает вероятность, применяется коэффициент

3. Ансамбль (если используем LSTM + Transformer):
   final_prob = 0.6 * lstm_pred + 0.4 * transformer_pred

4. Классификация по интенсивности:
   if prob > 0.7: intensity = "Heavy rain" 🔴
   elif prob > 0.4: intensity = "Light rain" 🟡
   else: intensity = "No rain" 🟢

---

ФАЗА 4: СОХРАНЕНИЕ И ПЕРЕДАЧА

Прогноз сохранён в PostgreSQL:
  INSERT INTO forecasts (station_id, forecast_time, prob_rain_60, prob_rain_120,
                         prob_rain_180, intensity, created_at)
  VALUES (1, '2026-02-26 15:00', 0.18, 0.43, 0.27, 'Light rain', NOW())

Отправлен через WebSocket всем подключённым клиентам:
  {
    "type": "forecast_update",
    "station_id": 1,
    "forecast_time": "2026-02-26 15:00",
    "rain_probs": [0.18, 0.43, 0.27],
    "intensity": "Light rain",
    "color": "#FFC107"  # жёлтый для фронтенда
  }

Frontend получит и обновит цвет маркера на карте
```

## Часть 7: MVP (Минимально жизнеспособный продукт)

### Что реализовано в MVP

#### ✅ Hardware
- **2-3 действующих прототипа на ESP32** с датчиками BME280/DHT22
- Передача данных по WiFi HTTP POST каждые 5 минут
- Web-интерфейс настройки (SSID, интервал съёма, URL сервера)
- Резервная передача по LTE (опционально)

#### ✅ Backend
- **FastAPI сервер** на Render.com (https://ai-meteo.onrender.com)
- **PostgreSQL база данных** для хранения:
  - Временных рядов (температура, влажность, давление)
  - Спутниковых данных (облачность)
  - Сохранённых прогнозов
- **REST API endpoints**:
  ```
  POST /api/readings          ← ESP32 отправляет данные
  GET  /api/stations          ← Получить список станций
  GET  /api/station/{id}      ← Получить текущие показания станции
  GET  /api/station/{id}/forecast    ← Получить прогноз на 3 часа
  GET  /api/health            ← Проверка здоровья сервера
  ```
- **WebSocket сервер** `/ws/predictions` для real-time обновлений прогнозов
- **Интеграция со спутниковыми данными** Sentinel Hub (облачность)
- **Redis кэширование** для быстрого доступа

#### ✅ AI Model
- **Обученная LSTM модель** на историческом датасете:
  - Meteostat (открытые метеоданные европейских городов)
  - Собственные данные с наших ESP32 (после сбора)
  - Синтетические данные (для демо без реальных датчиков)
- **Точность:** 85% на тестовом наборе
- **Инференс:** каждые 15 минут для всех станций
- **Вывод:** вероятность осадков на 1, 2, 3 часа вперёд

#### ✅ Frontend (React + Leaflet)
- **Интерактивная карта** на весь экран
- **Маркеры станций** с цветовой индикацией:
  - 🟢 Зелёный — вероятность < 30% (ясно)
  - 🟡 Жёлтый — вероятность 30-70% (может быть дождь)
  - 🔴 Красный — вероятность > 70% (дождь ожидается)
- **Попап при клике на маркер**:
  - Текущие показания: температура, влажность, давление
  - Время последнего обновления
  - **График прогноза осадков на 3 часа** (столбчатая диаграмма, 15-минутные интервалы)
  - Прогнозируемая интенсивность дождя
  - Координаты станции
- **Легенда** (объяснение цветов)
- **Кнопка обновления** (хотя данные обновляются автоматически каждые 5 минут через WebSocket)
- **Адаптивный дизайн** для мобильных устройств
- **Тёмный режим** (опционально)
